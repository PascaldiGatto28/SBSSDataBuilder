Games Results, Team and Players Table Document Help

In this page, we provide documentation for all the tables that appear in the actual Games, Team and Players page. Actual data is used, but just for league to make navigation easier. Just click "Show me" button to open text describing the associated table while viewing the table itself.

Data Store

The Data Store Information table is not really about game, team or player data, but describes the state of the file where all information is store (the data store). It is updated every week night at 8:30 PM. This table gives you an idea of the volume of data and how it is changing.
Most of the fields are obvious, but note that the number of players, like the number of teams changes as the season progresses, because the values are determined from the actual played games. For example, the number of players is the number of all players that have been in at least one game in any league, that is, it's not just roster players. Unless there are cancellations, the number of teams doesn't change after a completed full week's schedule.

Game Results

The Game Results Data for the 18 Monday Community Games table with all its nested tables encapsulates every bit on information that is in the data store. It also displays data calculated from the data. It is game-centric meaning the row of the table has all the information for every games played. There's a row for each game. Scheduled Game

The first cell is the scheduled game information, which is determined before the first games is played. If a game has not been played, or has been cancelled it does not appear in the table — after all, there's no data!

Teams and Players

The second cell in the row, for example, Teams and Players for the Ann Fraley-Long Realty vs Ambient Air Heating & Cooling Game, actually encapsulates the results. Although these results are posted on the SSSA web site, the values for team (and game outcome) are computed from the player "box line" values. This table obvious only has two row, one for each team. For each team, the data for each player is last cell of the row in this nested table. The rows are ordered as visitor's team then home team, not by outCome.

Players for the Visitors

Now we're getting to heart of the data: It's all about player stats. You'll notice that the table, for example, Players for Ann Fraley-Long Realty, is much like what appears in SSSA web site when you on a "Schedules" menu item to look the results a played game. There are some differences, however. All the usual calculated stats (average, slugging, etc.) are present. No question, that for a single game, this data not very interesting or useful, but to keep our tables consistent included those. More interesting a new stat is added: Plate appearances (PA) which is just the number offical at bats plus bases on balls and scrifice flies. The other difference which is useful is a summary line at the bottom that includes all player data and calculated stats.

Players for the Home Team

The second cell in the second row is just like the previous is for the home team rather than the visitors.

Team Summary Data & Standings

The second row in the main table, for example Team Summary Data and Standings for the 4 Monday Community Teams turns our perspective from games played to team performance. When the table is opened, a list of the teams is league is presented in "standings" order (actually, by wins and then by run differencial). You may notice, for example, that although "Andy's Irrigation-Andy McInnis" and "Barron Electric" have the same record, the former is in first place because it's run differencial (runs scored less runs against) is 25 versul -3 for "Barron Electric". (Full Disclosure: I do not know that is how the SSSA handles tie won/loss records).

Team Players Summary

The last cell in each row is (yet) another table, but this time is a cummulative summary of all players that played games for the team. The players that played 1,2 or 3 games like substitue players. But still they do contribute to the overall team stats, which are highlighted at the bottom of the table. Again, the line items for each player are the same when looking at single game, but in this casem the number games in which each player played is also presented. This data helps show the contribution players made to success of their teams.

Player Summary Stats

In this last table, for example, Player Summary Stats for the 59 Players and Rankings for 47 Players With Enough Plate Appearances for All Teams we really focus on the player. Here you see summary starts for every player in the league for all their activity on all teams. Because it's in a single, league it is reasonable to compare the stats. The player rows are sorted games played and then plate appearances. You open the last cell of row to see a table of the ranking of the player for the four standard computed stats &mdash Average (Avg), Slugging (Slug) Onbase percentage (OB%), and Onbase percentage plus slugging (OB%+Slug). Unlike the stats per game, these have real meaning.
Rather than opening the Ranking tables, just hover over the player name (second column) to see a pop-up of their photo and rankings. You can do that now, or ...
To calculate the rankings, a player must have had a minimal number of plate appearances. We choose, but the clearly subjective. The ranking are only based upon players that are qualified (that, have had enough plate appearances). Finally the last row is summary for all players and also the ranking of those values compared to all the qualified players. You would expect that the league ranks would be about in the middle (that is, about the median).
