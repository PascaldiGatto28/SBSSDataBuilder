Version History (Back to SBSS Data Viewer)

Version 1.15.24244, Released Date Saturday, August 31 2024
Two big items in this release: (1) All tables having player stats are now sortable, and (2) documentation for all SBSS Data Viewer table features and properties are documented in the About Tables page. A link to the page is now the first item in the Table of Contents on the home page, and is a must read (and entertaining).
  • Page titles (the text that appears in the browser tab) have been reviewed and shortened where possible.
  • Text to describe how to change the active season has been added to the home page.
  • Now that we have a page that documents Data Viewer tables, we removed any redundant text for other pages, including the home page. It helps keep pages focused on the topic content.
  • The page that describes items you should be aware when using SBSS Data Viewer has been renamed Things to Note & Information and the contents updated. All links to this page have been updated.
  • The usual number (too large) of minor text changes and some HTML formatting have been completed.
Version 1.12.24204, Released Date Monday, July 22 2024
The Summer 2024 is in full activity now, so the default season is now changed from the Spring 2024 to the Summer 2024, but remember you can select any season you want using the "Seasons Button" on the home page. There are now five seasons available. Unlike our release last week, this is a big one and has a significant (and I hope useful) new feature
  • Yet another a page! "Players Leagues Stats & Statistics", contains tables for every league and player summary stats and statistics for each of the leagues. The center piece is the player stats table that is sortable on every stat, statistic and player name. The sortable table is in response to suggestions from our visitors to the SBSS Data Viewer Web site. Even if you don't want to view the full statistics, hovering over the player's name displays their photo with a summary of the data. If you're into statistics, there is enough data on the page to calculate player z-Scores if you don't believe me (which you definitely should not)!
  • Button styling has changed to show different shade of color when hovering over or clicking on the button. The buttons (except the "ShowMe" buttons on the "guide" pages) also have tool tips when you hover over them.
  • The home page has been restyled a bit, added the link to the new page, and added information for sortable tables.
  • When you click on link to one the "Guide" pages, the guide is opened in the same tab to make it easier to return to the home page or the original page.
  • Lots of small changes to improve textual content and styling. If you don't notice, I guess that's a good thing.

  • Spoiler Alert — Coming soon! A new page summarizing League and Player stats and statistics for all players for each league is nearing completion. The tables containing player data are sortable by any stat. Later player tables on other pages will be sortable. If you'd like to be notified when new features and changes are available, send me an email at Richard.Levaro@outlook.com and check out Feedback & Contact Information.
Version 1.12.24164, released Date Wednesday, June 12 2024
Lots of little things, but the big (no, huge) thing is that Z-scores are now included in the Player Sheets.
  • Implementing Z-scores required building a lot of infrastructure to build and display the statistic. You'll notice that, of course, in the Player Sheets (and Guide to) data page. It has been worked into the tables the way we display percentiles and rankings. The other place is in the Definitions, Explanations & Details page. I hope you find that useful and interesting.
  • Speaking of the statistics, we removed the link to "Lies, Damned Lies and Statistics" because it's still empty, but the "Definitions, Explanations & Details" page gives a glimpse of what we're working on next. We expect to have graphs and plots for all the leagues in the next release, sometime during next season. We really do need to get some feedback to help us to spend the time with what you find important.
  • In general, the table of contents of the main page has been altered to reflect use cases. The notice at the top of the page has been removed, but the link will not be remove of course.
  • A whole lot of text changes, a few simple bug fixes also completed. Not every bug is a feature, but some are and some bugs are features. Sounds of Yogi Berra.
Version 0.11.24140, released Date Sunday, May 19 2024
Just a few bug fixes and a link to the "Lies, Damned Lies and Statistics". Nothing there yet, but working on it now, and coming soon.
  • The program that checks for changes in the game schedules (played, canceled, etc.) now runs at 6:20 PM rather than at 8:30 PM on every weekday.   • The "Player Summary Stats" for a team in the "Game Results, Teams and Players" page, did not expand the "Rankings" table properly. It now expands and collapses that table properly. Published on May 18, 2024.

Version 0.10.24122, released Wednesday, May 1 2024
  • The biggest (and most important) feature for this release in the introduction of percentiles and ranks on the Player Sheets page. This allows to see summary data for any active player, their rank and percentiles for each of AVG, SLG, OB, OPS stats. Be sure and check the Definitions, Explanations & Details page for information about these additional stats, how they're used and calculated. This change required many other supporting changes as well.
  • The Spring 2024 season has been added to the drop-down list to choose the season data you want to look at. It'll take a week after play starts for every league to have data. If you look at a league with no activity, you won't see any data, but you'll the normal introductory text. Because there won't be data for a week or two, the Winter 2024 is still default, but we'll change that once play begins. Remember that Log Sessions and the Data Store Information always use the current season which now is Spring 2024. There is an entry the Log Sessions entry which shows the data store has been created. The Data Store Information page shows the number of leagues (8) and the number of scheduled games (64), but essentially everything else is zero.
  • Lots of small changes to text and HTML to the "Data Store Information" to support the existence of forfeited games. These are things that are just from the inclusion of a new category of games "played" that I forgot about.
  • The release notes have been split between this page and "their own" – Release Notes. The latest release will be here, but all release items can be found in the "Release Notes", the link for which appears in the contents and the title of this section.
  • Some text changed to improve clarity and tie definitions with abbreviations in the Definitions, Explanations & Details page.
  • The message at the top of the home page asking to you to read "Notes & Information" has been dialed back to be a little more subtle, and a typo was fixed.
  • Log Sessions are back. This is kind of an IT thing (Information Technology — keeping the wheels on the engine), but you might find it entertaining.
Version 0.10.24112, released April 21, 2024
  • On the last day of 2014 Winter games, we had a forfeited game which I didn't process properly from the update program. That has been fixed. It's unique because the game in theory and has been played, but in practice there is no player data. To quote an expert in such matters (Yogi Berra): In theory there is no difference between theory and practice — in practice there is." Practically speaking, the 7 runs allocated to the team that did not forfeit, and the number wins and losses reflect the score in the team summary. But since there's no player data, I do not display fictitious game player stats. Take a look at the Friday Competitive Winter 2014 league from the Game Results, Teams and Players, you'll see 10 games of player data, but 11 games in the game overview. Always trust the "Yogi."
  • This was mentioned in the previous release notes, but as reminder, the season cookie that keeps track of your season selection is discarded when all instances of the browser you used are closed. In that case, when you reenter the SBSS Data Viewer Web site, the default season (current or latest) is selected.
  • Minor changes to the Things to Note & Information page to reflect changes to the use of a single discardable cookie.
  • Changed this release notes section so that only the latest changes are visible unless you open the rest of the notes using the "More Release Notes ↓ ..." button (see below).
  • The Feedback & Contact section has been expanded to include how emails to those interested in receiving notification of updates and changes are sent. Please take a look.
